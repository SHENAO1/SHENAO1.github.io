用 FFT 计算互相关：共轭、频域乘法和反向索引的区别

这份笔记默认讨论复数离散序列。实数序列只是它的特例；一旦序列带 I/Q 分量、复基带或相位信息，复共轭放在哪里就会直接改变结果。

1. 先定互相关方向

这里采用下面这个方向定义：

\[ r_{ab}[m]=\sum_n \overline{a[n]}\,b[n+m] \tag{1} \]

这个式子的读法是：把 \(a\) 当作参考模板，在 \(b\) 里面向右移动 \(m\) 个采样去匹配。若 \(b\) 是 \(a\) 的延迟版本，相关峰通常出现在对应的正延迟附近。

不同软件库可能把输入顺序、lag 符号或输出排列定义得不一样，所以先写清楚公式，比记某个函数名更可靠。

参考序列 \(a[n]\) 被共轭，表示拿它作为匹配模板。
被搜索序列 \(b[n+m]\) 带延迟，表示在 \(b\) 中寻找相似位置。
峰值位置 lag 的正负由定义决定，不能脱离公式单独判断。

2. 标准 FFT 写法

对长度为 \(N\) 的循环相关，设：

\[ A[k]=\operatorname{FFT}\{a[n]\},\qquad B[k]=\operatorname{FFT}\{b[n]\} \tag{2} \]

那么式 (1) 对应的循环互相关可以写成：

\[ r_{ab}[m] = \operatorname{IFFT}\left\{ \overline{A[k]}\,B[k] \right\} \tag{3} \]

代码里就是：

r_ab = ifft(conj(fft(a)) .* fft(b));

关键点是：先把 \(a\) 做 FFT 得到 \(A[k]\)，再对频谱 \(A[k]\) 取共轭。这个共轭来自相关定理，而不是把 \(a[n]\) 先改成 \(\overline{a[n]}\)。

3. 为什么 fft(conj(a)) 不等价

很多错误就藏在这一行：

ifft(fft(conj(a)) .* fft(b))

它看起来像“也对 \(a\) 做了共轭”，但频域上并不等于式 (3)。DFT 有一个重要性质：

\[ \operatorname{FFT}\{\overline{a[n]}\}[k] = \overline{A[(-k)\bmod N]} \tag{4} \]

也就是说，时域先共轭，会让频谱不只是共轭，还会把频率索引反向。于是：

\[ \operatorname{IFFT}\left\{ \operatorname{FFT}(\overline{a})\operatorname{FFT}(b) \right\} = \overline{a[n]} \circledast b[n] \tag{5} \]

这里 \(\circledast\) 表示循环卷积。它算的是“\(\overline{a}\) 与 \(b\) 的卷积”，不是式 (1) 的互相关。

一句话区分

互相关需要“共轭频谱相乘”；卷积需要“两个频谱直接相乘”。如果你写的是 fft(conj(a)) .* fft(b)，它更像卷积路径。

4. 交换 a 和 b 的关系

如果换成 \(b\) 对 \(a\) 的相关：

\[ r_{ba}[m] = \sum_n \overline{b[n]}\,a[n+m] = \operatorname{IFFT}\left\{ \overline{B[k]}\,A[k] \right\} \tag{6} \]

它和 \(r_{ab}\) 的关系是：

\[ r_{ba}[m]=\overline{r_{ab}[-m]} \tag{7} \]

所以两者不是完全相同的数组。若序列是复数，交换输入会同时带来两个变化：lag 方向反过来，结果还要取复共轭。

如果只关心相关峰的幅度，\(|r_{ab}[m]|\) 和 \(|r_{ba}[-m]|\) 对应相等；但如果还要看相位，二者不能随意混用。

5. 积再取 FFT 是什么

如果真的写成：

fft(fft(conj(b)) .* fft(a))

那最后一步不是互相关公式所需的 IFFT。按照 DFT 乘积定理，中间乘积仍对应某个循环卷积；最后再做一次 FFT，只会得到这个卷积结果的缩放和索引重排版本，而不是把它变回相关。

因此，常规相关的最后一步应是：

ifft(conj(fft(reference)) .* fft(signal))

如果你看到“乘完再 FFT”，通常要先判断是不是笔误；除非整个推导明确在频域里又定义了一次新的变换，否则它不属于普通相关计算链路。

6. 循环相关与线性相关

直接用长度为 \(N\) 的 FFT，会得到循环相关：

\[ r_{ab}^{(c)}[m] = \sum_{n=0}^{N-1}\overline{a[n]}\, b[(n+m)\bmod N] \tag{8} \]

这里的 \(\bmod N\) 会让尾部绕回头部。如果你要普通线性相关，就需要把 FFT 长度扩到：

\[ L\ge N_a+N_b-1 \tag{9} \]

然后使用：

\[ r_{ab}^{(lin)} = \operatorname{IFFT}\left\{ \overline{\operatorname{FFT}(a,L)}\, \operatorname{FFT}(b,L) \right\} \tag{10} \]

需要注意输出排列：IFFT 结果前半段通常是非负 lag，末尾一段是负 lag。若要按 lag 从负到正显示，需要把末尾的负 lag 部分搬到前面。

7. MATLAB / NumPy 对照

下面的代码按式 (1) 的定义输出线性相关，并把 lag 排列成从负到正：

MATLAB

a = [1+2j, 2-1j, -1+0.5j];
b = [2-1j, -0.5+1j, 3+0j, 1-2j];

na = numel(a);
nb = numel(b);
L = na + nb - 1;

rab_circ = ifft(conj(fft(a, L)) .* fft(b, L));
lags = [-(na-1):-1, 0:(nb-1)];
rab = [rab_circ(end-na+2:end), rab_circ(1:nb)];

rba_circ = ifft(conj(fft(b, L)) .* fft(a, L));
rba = [rba_circ(end-nb+2:end), rba_circ(1:na)];

NumPy

import numpy as np

a = np.array([1+2j, 2-1j, -1+0.5j])
b = np.array([2-1j, -0.5+1j, 3+0j, 1-2j])

L = len(a) + len(b) - 1
rab_circ = np.fft.ifft(np.conj(np.fft.fft(a, L)) *
                       np.fft.fft(b, L))

neg = rab_circ[-(len(a)-1):] if len(a) > 1 else np.array([])
rab = np.concatenate([neg, rab_circ[:len(b)]])
lags = np.arange(-(len(a)-1), len(b))

如果只想做循环相关，把 FFT 长度固定为周期长度即可；如果要线性相关，就必须用 \(L\ge N_a+N_b-1\) 的零填充长度。

8. 一页总结

写法 含义 是否是本文定义的 \(r_{ab}\)
ifft(conj(fft(a)) .* fft(b)) \(a\) 作参考、\(b\) 作被搜索序列的互相关。 是
ifft(conj(fft(b)) .* fft(a)) \(b\) 作参考、\(a\) 作被搜索序列的互相关。 不是同一个方向，结果共轭反向
ifft(fft(conj(a)) .* fft(b)) \(\overline{a}\) 与 \(b\) 的循环卷积。 不是
fft(fft(conj(b)) .* fft(a)) 乘积后又做 DFT，属于额外变换。 不是常规相关

最稳的记法是：相关等于“参考信号频谱取共轭，再乘被搜索信号频谱，最后 IFFT”。若要交换参考和被搜索对象，就同时准备好接受共轭和 lag 反向。
