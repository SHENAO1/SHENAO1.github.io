第8讲： 循环前缀与加窗——降低带外泄漏

引言

上一讲我们聊了 IFFT✦ 的硬件实现，搞清楚了如何把频域数据高效地变成时域的 OFDM 信号。那么现在信号已经生成了，接下来就面临一个”看似简单、实则棘手”的工程问题——带外泄漏。

什么是带外泄漏？简单说，就是你的信号跑到了别人的”地盘”上。无线通信里频谱是稀缺资源，运营商花了几十亿买来的一块频谱，如果信号跑出去干扰隔壁频段，那就是违反监管规定的大事。3GPP 对每个频段都有严苛的频谱模板✦（Spectrum Emission Mask）要求，信号功率在带外必须衰减到规定值以下。

造成带外泄漏的主要原因，出乎意料地简单：矩形脉冲✦。

这一讲，我们就来彻底搞清楚：矩形脉冲为什么旁瓣高？加窗如何解决这个问题？以及5G NR是怎么配置 CP 和加窗的？

1. 矩形脉冲与频谱旁瓣：问题的根源

1.1 OFDM 符号的时域形态

一个标准的OFDM符号由两部分组成：循环前缀✦（CP）+ 有效数据段。加入 CP 之后，时域信号大致如下：

|<--- CP (Ncp samples) --->|<-------- 有效数据 (N samples) --------->|
|        复制自数据末尾     |          IFFT输出                       |
|__________________________|_________________________________________|
        CP区域                         数据区域

在时域上，这段信号被一个矩形窗截断——信号突然从 0 开始、突然截止。这个突变就是带外泄漏的元凶。

1.2 矩形窗的频域特性

一个长度为 $N$ 的矩形窗，其频域响应是 sinc 函数：

\[ W_{\text{rect}}(f)=\frac{\sin(\pi fNT_s)}{\pi fT_s}e^{-j\pi f(N-1)T_s} \]

sinc 函数的旁瓣衰减极其缓慢：

旁瓣编号 距主瓣距离 旁瓣峰值 相对衰减
第1旁瓣 $1.5/(NT_s)$ -13.3 dB 仅 13 dB
第2旁瓣 $2.5/(NT_s)$ -17.8 dB 仅 18 dB
第5旁瓣 $5.5/(NT_s)$ -26.3 dB 仅 26 dB

而3GPP要求的带外衰减往往需要 -30 dB 乃至 -50 dB 以上，矩形窗远远不够。

2. 加窗：让边缘平滑

2.1 加窗的核心思想

加窗的本质是：把时域信号的起始和结束边缘从”硬切断”变成“软过渡”。原来信号是
1 → 1 →... → 1 → 0（矩形），加窗后变成
0 → 0.2 → 0.7 → 1 → 1 →... → 1 → 0.7 → 0.2 → 0（平滑）。

代价： 加窗会让 OFDM 符号的”上升沿”和”下降沿”变长，如果处理不当会引入符号间干扰（ISI）。

解决方案： 利用 CP 的冗余来”藏”住过渡段，过渡段与相邻符号的 CP 部分重叠，不占用额外时频资源。

2.2 升余弦（Raised Cosine）窗

最常用的窗函数之一是升余弦窗✦，其过渡段的形状由余弦曲线定义：

\[ w[n]= \begin{cases} \dfrac{1}{2}\left(1-\cos\dfrac{\pi n}{\beta}\right), & 0\le n<\beta \\ 1, & \beta\le n<N+N_{cp}-\beta \\ \dfrac{1}{2}\left(1+\cos\dfrac{\pi(n-N-N_{cp}+\beta)}{\beta}\right), & N+N_{cp}-\beta\le n\le N+N_{cp} \end{cases} \]

其中 $\beta$ 是过渡段长度（单位：采样点），通常取 $\beta=0.1\times N_{cp}$ 到 $0.25\times N_{cp}$。

2.3 各种窗函数的频域性能对比

除了升余弦窗，工程中还常用以下窗函数：

窗函数 主瓣宽度 第一旁瓣 旁瓣衰减速率 典型应用
矩形窗 $2/N$ -13.3 dB -6 dB/oct 无加窗
汉宁窗 $4/N$ -31.5 dB -18 dB/oct 通用场景
汉明窗 $4/N$ -41.0 dB -6 dB/oct 旁瓣一致性好
升余弦窗 $\sim3.5/N$ -23 dB -18 dB/oct OFDM 加窗首选
高斯窗 可变 -55 dB+ 极快 对主瓣宽度不敏感场景
Blackman窗 $6/N$ -57.5 dB -18 dB/oct 高衰减需求

对于 OFDM 系统，升余弦窗是首选：它的过渡段与 CP 重叠实现方式天然契合，实现简单，频谱模板表现良好。

2.4 MATLAB仿真：加窗前后频谱对比

%% MATLAB: OFDM加窗前后频谱对比
clear; clc;

%--- 参数设置 ---
N     = 2048;      % FFT点数（LTE 20MHz）
Ncp   = 144;       % CP长度（长CP，首符号）
beta  = 32;        % 加窗过渡段（采样点数）

%--- 生成随机OFDM符号 ---
X = (randn(N,1) + 1i*randn(N,1)) / sqrt(2); % 随机频域数据
x = ifft(X);                                % IFFT

%--- 添加循环前缀 ---
x_cp = [x(end-Ncp+1:end); x];  % 长度 N + Ncp

%--- 矩形窗（不加窗） ---
x_rect = x_cp;

%--- 升余弦窗 ---
win = ones(N+Ncp, 1);
% 上升沿（利用CP区域）
win(1:beta) = 0.5 * (1 - cos(pi*(0:beta-1)'/beta));
% 下降沿
win(end-beta+1:end) = 0.5 * (1 + cos(pi*(0:beta-1)'/beta));

x_win = x_cp .* win;

%--- 计算功率谱 ---
NFFT = 16384;
Prect = 20*log10(abs(fftshift(fft(x_rect, NFFT))));
Pwin  = 20*log10(abs(fftshift(fft(x_win, NFFT))));

%--- 归一化到最大值 ---
Prect = Prect - max(Prect);
Pwin  = Pwin - max(Pwin);

%--- 绘图 ---
f = linspace(-0.5, 0.5, NFFT);
figure; hold on;
plot(f, Prect, 'b', 'LineWidth', 1, 'DisplayName', '矩形窗');
plot(f, Pwin, 'r', 'LineWidth', 1.5, 'DisplayName', '升余弦窗');
ylim([-80 5]); xlim([-0.7 0.7]);
xlabel('归一化频率'); ylabel('功率谱 (dB)');
title('OFDM加窗前后频谱对比'); legend; grid on;

运行上述代码，你会看到一个典型结果：在带外频率偏移约1.5倍子载波间隔处，升余弦窗相比矩形窗能够提供约 20~25 dB 的旁瓣抑制。

3. EVM 与频谱模板的权衡

加窗是“双刃剑”，用得好解决泄漏问题，用过头会损害 EVM。

3.1 加窗对EVM的影响

窗函数施加在时域信号的边缘，理论上不影响有效数据区间的 EVM。但实际中有两个问题：

问题1： 过渡段对相邻符号的污染

如果窗的上升沿和下降沿超出 CP 范围，就会侵入前一个或后一个符号的有效数据区间，引入 ISI，直接拉高 EVM。

设计约束：

\[ 2\beta\le N_{cp} \]

过渡段总长度不得超过 CP 长度。这是加窗设计的铁律。

问题2： 多径信道✦下的影响

在多径信道中，由于信号存在时延扩展，CP能抵御的最大时延为 $N_{cp}$ 个采样点。加窗后，实际可用的CP”保护区间”缩短为 $N_{cp}-2\beta$，因此：

\[ \tau_{\max}\le (N_{cp}-2\beta)\cdot T_s \]

实际设计建议：

  • $\beta$ 取 $N_{cp}$ 的 10%~15% 是较为保守的安全选择
  • 对于短CP（$N_{cp}=72$，LTE普通符号），$\beta=8\sim 10$个采样点即可
  • 对于长CP（$N_{cp}=512$，LTE特殊子帧），$\beta$ 可放宽至 40 ~ 60

3.2 EVM vs. 带外抑制的设计空间

高 | •矩形窗（EVM好，带外差）
   |
带 |          •升余弦窗（β=Ncp/8）
外 |
抑 |                 •升余弦窗（β=Ncp/4）
制 |
低 |                        •汉宁窗（EVM受损，带外最好）
   |____________________________
    好        EVM表现        差

工程中，升余弦窗 + $\beta=N_{cp}/8$ 这个参数点是性价比最高的选择，既能满足3GPP频谱模板，又不会明显劣化EVM。

4. 5G NR中的CP与SCS配置

4.1 灵活参数集✦（Flexible Numerology）

5G NR引入了灵活参数集（Numerology），子载波间隔（SCS）可以是 15/30/60/120/240 kHz，对应关系为：

\[ \Delta f=2^\mu\times 15\,\text{kHz},\qquad \mu\in\{0,1,2,3,4\} \]

不同 SCS 下，OFDM 符号时长和 CP 长度也随之缩放：

SCS $\Delta f$ $\mu$ 符号时长 $T_u$ 普通 CP $N_{cp}$ 长 CP $N_{cp}^{(E)}$ 典型场景
15 kHz 0 66.67 μs 4.69 μs
(144采样)		 5.21 μs
(160采样)		 eMBB低速
30 kHz 1 33.33 μs 2.34 μs
(144采样)		 2.60 μs
(160采样)		 eMBB高速
60 kHz 2 16.67 μs 1.17 μs
(144采样)		 — eMBB超高速
120 kHz 3 8.33 μs 0.586 μs — mmWave
240 kHz 4 4.17 μs 0.293 μs — 参考信号专用

注意： SCS翻倍时，时域所有参数（符号时长、CP时长）均减半，但采样点数不变——因为带宽也翻倍了，采样率同步翻倍。

4.2 5G NR中的加窗实现

5G NR规范（3GPP TS 38.104）对带外辐射有严格要求，实际设备中会结合以下手段：

  1. CP加窗：如前所述，利用 CP 边缘做升余弦过渡
  2. 数字滤波：发射滤波器对整个带宽做成形滤波（Root-Raised Cosine✦ 或 FIR滤波）
  3. 功放线性化：通过数字预失真（DPD）减小非线性造成的带外泄漏

其中 CP加窗 是最轻量的方案，不增加额外延迟，实现简单，是5G基站基带芯片的标配。

4.3 与LTE加窗的对比

特性 LTE 5G NR
SCS 固定 15 kHz 灵活 15/30/60/120 kHz
CP长度 普通 4.69 μs / 长 5.21 μs 随SCS缩放
加窗策略 可选 强烈推荐（频谱模板更严格）
带外要求 $-30\sim -45\,\text{dBc}$ $-35\sim -55\,\text{dBc}$
典型 $\beta$ 8~16 采样点 4~8 采样点（高SCS下）

5. 工程实现要点

5.1 发射侧加窗的实现流程

频域数据 X[k]
      |
      ▼
   IFFT （N点）
      |
      ▼
添加循环前缀 → x_cp[n]（长度 N + Ncp）
      |
      ▼
乘以升余弦窗 → x_win[n] = w[n] × x_cp[n]
      |
      ▼
符号拼接（相邻符号的过渡区域重叠相加）
      |
      ▼
DAC → 射频前端

5.2 符号拼接时的重叠相加

加窗后的符号不能简单地首尾拼接，而需要”重叠相加（Overlap-Add）”：

%% 符号拼接（Overlap-Add）实现
% 假设两个连续OFDM符号已加窗: sym1_win, sym2_win
% beta = 过渡段长度，overlap区域在符号末尾/开头各beta个采样

output_len = N + Ncp + beta;  % 每符号实际占用的时间槽

% 拼接时，sym1末尾beta个点 与 sym2开头beta个点 相加
frame(1 : N+Ncp)             = sym1_win(1 : N+Ncp);  % sym1主体
frame(N+Ncp-beta+1 : end)   = frame(N+Ncp-beta+1:end) + sym2_win(1:beta);  % 重叠相加

这个操作确保了边缘平滑过渡，同时不引入额外的时频资源占用。

总结

这一讲我们深入剖析了 OFDM 带外泄漏的根源和解决方案：

  1. 问题根源： 矩形脉冲的 sinc 频域响应旁瓣衰减仅 13 dB，无法满足无线通信频谱模板要求
  2. 加窗原理： 在 OFDM 符号边缘施加平滑过渡，将硬截断变为软过渡，有效压制旁瓣
  3. 升余弦窗： OFDM 系统的首选窗函数，过渡段自然嵌入 CP 区间，不额外消耗资源
  4. 设计约束： 过渡段长度 $2\beta\le N_{cp}$，且需为多径保留足够的 CP 保护
  5. 5G NR 实践： 灵活 SCS 下加窗参数随符号时长动态调整，结合数字滤波满足更严苛的频谱模板
