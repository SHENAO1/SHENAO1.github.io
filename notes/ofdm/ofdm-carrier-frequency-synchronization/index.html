第13讲：载波频率同步——消除频偏的影响

一、CFO从哪里来？

载波频偏（Carrier Frequency Offset，CFO）是指接收机本振频率 $f_r$ 与发射机载波频率 $f_c$ 之间的差值：

\[ \Delta f=f_r-f_c \]

实际系统中，CFO主要来自两个来源：

1. 晶振（本振）频率偏差

无论是发射机还是接收机，本振都由石英晶振✦产生。受工艺误差、温度变化等影响，晶振频率不可避免地存在偏差，典型量级用 ppm（百万分之一） 表示。

例如，LTE系统中：

  • 基站本振精度要求：$\le \pm 0.05\,\mathrm{ppm}$
  • 终端本振精度要求：$\le \pm 0.1\,\mathrm{ppm}$

在2.6 GHz（Band 38）载波频率下，0.1 ppm对应的绝对频偏为：

\[ \Delta f=2.6\times 10^9\times 0.1\times 10^{-6}=260\,\mathrm{Hz} \]

而LTE 15 kHz子载波间隔意味着这一频偏已达子载波间隔的 $260/15000\approx 1.7\%$，如果是收发两端误差相叠加，总偏差可达数百Hz量级。

2. 多普勒频移✦

当发射机或接收机处于运动状态（如高速列车、车载通信）时，会产生多普勒频移：

\[ f_d=\frac{v}{c}\cdot f_c \]

以速度 $v=120\,\mathrm{km/h}\;(\approx 33.3\,\mathrm{m/s})$，$f_c=2.6\,\mathrm{GHz}$为例：

\[ f_d=\frac{33.3}{3\times 10^8}\times 2.6\times 10^9\approx 288\,\mathrm{Hz} \]

同样是十几个kHz级别的子载波间隔面前，这已经是相当显著的频偏。

二、CFO如何破坏OFDM的正交性

这是本讲最核心的内容。我们来定量分析CFO对接收信号的影响。

设发送的第 $k$ 个子载波符号为 $X_k$，经过无噪声 AWGN 信道后，接收端时域信号（去除CP后）为：

\[ r(n)=\sum_{k=0}^{N-1}X_k\cdot e^{j2\pi\frac{k}{N}n}\cdot e^{j2\pi\frac{\Delta f}{f_s}n}, \qquad n=0,1,\ldots,N-1 \]

其中 $f_s$ 为采样率，$\varepsilon=\Delta f/\Delta f_{sc}$ 为归一化频偏（以子载波间隔为单位，$\Delta f_{sc}=f_s/N$）。

接收端对第 $l$ 个子载波做FFT解调：

\[ \hat{X}_l=\frac{1}{N}\sum_{n=0}^{N-1}r(n)\cdot e^{-j2\pi\frac{l}{N}n} \]

代入 $r(n)$ 并化简（令 $m=k-l+\varepsilon$）：

\[ \hat{X}_l= X_l\cdot \underbrace{\frac{\sin(\pi\varepsilon)}{N\sin(\pi\varepsilon/N)} e^{j\pi\varepsilon(N-1)/N}}_{\text{信号幅度衰减 + 公共相位旋转（CPE）}} +\underbrace{\sum_{k\ne l}X_k\cdot \mathrm{ICI}_{kl}}_{\text{子载波间干扰（ICI）}} \]

这个表达式揭示了CFO的两重影响：

①公共相位误差（CPE，Common Phase Error）

每个子载波上的信号都乘以相同的复数相位因子，导致整个星座图发生同向旋转。CPE与OFDM符号序号成比例增加，表现为星座图的整体旋转。

\[ \mathrm{CPE}_m=e^{j2\pi\varepsilon m},\qquad m\text{ 为符号序号} \]

②子载波间干扰（ICI，Inter-Carrier Interference）

CFO破坏了子载波之间的正交性，导致相邻子载波的能量“泄漏”到目标子载波上。当 $\varepsilon=0$ 时，ICI项完全消失；当 $\varepsilon$ 增大时，ICI迅速增强。

ICI的信噪比损失可用如下近似公式估算：

\[ \mathrm{SINR}\approx \frac{3}{(\pi\varepsilon)^2} \]

例如 $\varepsilon=0.1$（频偏为子载波间隔的10%），SINR损失约为：

\[ \mathrm{SINR}\approx \frac{3}{(\pi\times 0.1)^2}\approx 30.4\,\mathrm{dB} \]

这个数字说明，只要频偏控制在子载波间隔的5%以内，ICI的影响就相对可接受；一旦超过10%，信号质量将显著恶化。

三、频偏估计算法

工程中的CFO估计分为两个阶段：粗估计（捕获阶段）和精估计（跟踪阶段）。

3.1 基于CP的粗估计

利用CP与符号末尾部分的相关性（见第12讲），不仅可以做定时同步，还可以提取粗频偏信息。

设CP长度为 $L_{cp}$，同一OFDM符号中CP部分与尾部的相关值为：

\[ P=\sum_{n=0}^{L_{cp}-1}r^*(n)\cdot r(n+N) \]

频偏估计：

\[ \hat{\varepsilon}_{\mathrm{rough}}=\frac{\angle P}{2\pi} \]

此方法的频偏估计范围限制在 $|\varepsilon|\le 0.5$（即半个子载波间隔），适用于粗估计。

3.2 Moose算法✦（训练符号法）

Moose算法（1994年提出）是频偏估计的经典方法，广泛应用于IEEE 802.11a/g/n WiFi系统。

基本思想： 发送两个完全相同的OFDM训练符号，利用它们在时域上的相位差推断频偏。

设两个相邻训练符号的时域序列为 $r_1(n)$ 和 $r_2(n)$（去CP后），频偏 $\Delta f$ 导致第二个符号相对第一个符号有额外相位旋转：

\[ r_2(n)=r_1(n)\cdot e^{j2\pi\Delta f T_{sym}} =r_1(n)\cdot e^{j2\pi\varepsilon} \]

其中 $T_{sym}=N/f_s$ 为FFT窗长。

Moose估计量：

\[ \hat{\varepsilon}= \frac{1}{2\pi}\angle \left( \sum_{n=0}^{N-1}r_1^*(n)\cdot r_2(n) \right) \]

Moose算法的估计范围为 $|\varepsilon|\le 0.5$，若频偏超过半个子载波间隔，需先做整数倍频偏检测再精估计。

对于大频偏场景（如多普勒严重的高速移动场景），可将训练符号分为更短的重复段，牺牲一些精度来换取更大的估计范围：

\[ \hat{\varepsilon}_{\mathrm{large}}= \frac{N}{2\cdot L}\cdot \frac{1}{2\pi}\angle \left( \sum_{n=0}^{L-1}r_1^*(n)\cdot r_1(n+L) \right) \]

其中 $L$ 为重复段长度，估计范围扩大到 $|\varepsilon|\le N/(2L)$。

3.3 基于导频的频偏跟踪

在帧数据传输阶段（无训练符号），利用已知位置的导频子载波持续跟踪残余频偏变化。

OFDM接收端每个符号后，导频子载波的相位旋转量为：

\[ \Delta\phi_m=2\pi\varepsilon\cdot m \]

其中 $m$ 为符号序号。通过连续符号的导频相位差分，可以估计并跟踪残余频偏：

\[ \hat{\varepsilon}_{\mathrm{track}}= \frac{\angle\left(P_{\mathrm{pilot}}^{(m)}\cdot P_{\mathrm{pilot}}^{*(m-1)}\right)}{2\pi} \]

这就是LTE/NR系统中相位跟踪参考信号（PT-RS✦）的核心思想，我们将在第20讲专门讨论。

四、频偏补偿实现

估计到频偏 $\hat{\varepsilon}$ 后，补偿分为两个层面：

4.1 时域频旋转（整体CFO补偿）

在FFT之前，对时域采样序列逐点乘以共轭相位项：

\[ \tilde{r}(n)=r(n)\cdot e^{-j2\pi\hat{\varepsilon}n/N} \]

这是最直接的补偿方式，计算量为每采样点一次复数乘法。

MATLAB实现如下：

function r_comp = cfo_compensate(r, epsilon_hat, N)
% r: 接收时域序列（去CP后，长度N）
% epsilon_hat: 归一化频偏估计值
% N: FFT长度

n = (0:N-1).';
phase_correction = exp(-1j * 2 * pi * epsilon_hat * n / N);
r_comp = r .* phase_correction;
end

4.2 频域相位补偿（CPE补偿）

时域补偿之后，若仍有残余CPE（来自相位噪声或残余CFO），可在FFT之后对各子载波做统一相位旋转：

\[ \hat{X}_k^{\mathrm{comp}}=\hat{X}_k\cdot e^{-j\hat{\phi}_{\mathrm{CPE}}} \]

其中 $\hat{\phi}_{\mathrm{CPE}}$ 通过已知导频子载波估计：

\[ \hat{\phi}_{\mathrm{CPE}}= \frac{1}{|\mathcal{P}|}\sum_{k\in\mathcal{P}} \angle\left(\hat{X}_k\cdot X_k^{*,\mathrm{pilot}}\right) \]

$\mathcal{P}$ 为导频子载波集合，$X_k^{\mathrm{pilot}}$ 为已知导频符号。

4.3 两级补偿的完整流程

时域接收信号 r(n)
        |
        ▼
[粗CFO估计] ← 基于CP相关 / 训练符号Moose算法
        |
        ▼ ε̂_rough
[时域粗补偿] r_comp(n) = r(n)·exp(-j2πε̂n/N)
        |
        ▼
[去CP] -> [FFT]
        |
        ▼
[导频提取] -> [精CFO估计 / CPE估计]
        |
        ▼ φ̂_CPE
[频域CPE补偿] X̂_k^comp = X̂_k · exp(-jφ̂_CPE)
        |
        ▼
[信道估计 -> 均衡 -> 解调]

五、5G NR✦中的CFO与相位噪声补偿

5G NR引入了更高的载波频率（毫米波，最高52.6 GHz）和更大的子载波间隔（最高240 kHz），CFO问题更加突出：

频段 子载波间隔 典型多普勒 @120km/h 归一化频偏 $\varepsilon$
FR1（低于6GHz） 15/30 kHz 288 Hz 0.019~0.010
FR2（毫米波，28GHz） 60/120 kHz 3.1 kHz 0.052~0.026
FR2（毫米波，52GHz） 120/240 kHz 5.8 kHz 0.048~0.024

可以看到，5G NR毫米波频段的归一化频偏仍然控制在5%以内，这也是5G选择更大子载波间隔的重要原因之一——让相同的绝对频偏对应更小的归一化频偏，从而降低ICI。

此外，毫米波本振的相位噪声更大（与载波频率的平方成正比），NR引入了相位跟踪参考信号（PT-RS）专门用于CPE的逐符号补偿，这部分将在第20讲详细讲解。

六、工程调试中的CFO问题

做过基带算法调试的同学对CFO一定不陌生。以下是几个实际工程经验：

经验1：先看星座图的旋转模式

  • 所有子载波星座点整体均匀旋转，且随时间呈线性相位增加 → 典型的残余CFO特征
  • 不同子载波旋转方向不一致，或旋转随子载波编号变化 → 可能是ICI或信道估计问题

经验2：频偏估计精度要求

一般要求残余归一化频偏 $|\varepsilon_{\mathrm{residual}}|<1\%$，对应SINR损失不超过约0.1 dB。对于高阶调制（256QAM），要求更严格，需达到0.1%以内。

经验3：高速场景下的频偏跟踪

在高铁（$v>200\,\mathrm{km/h}$）场景下，多普勒不仅带来静态CFO，还会导致频偏随时间漂移（符号间频偏变化）。此时单次估计不够，需要实现环路滤波器（类似PLL）对CFO进行连续跟踪，防止系统失同步。

总结

本讲核心要点：

  1. CFO来源：晶振频率偏差（ppm级）+ 多普勒频移，典型量级数百Hz
  2. CFO影响：公共相位误差（CPE，使整个星座图旋转）+ 子载波间干扰（ICI，破坏正交性），归一化频偏超过10%时SINR损失明显
  3. 估计算法：
  • 粗估计：基于CP自相关，范围 $\pm 0.5$ 子载波间隔
  • 精估计：Moose算法（训练符号），基于导频的跟踪
  1. 补偿方式：时域频旋转（粗补偿）+ 频域CPE补偿（精校正）
  2. 5G NR增强：更大子载波间隔降低归一化频偏，PT-RS专门补偿CPE
