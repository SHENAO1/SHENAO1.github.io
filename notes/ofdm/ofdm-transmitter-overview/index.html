一、完整的发射机链路长什么样

如果你打开任何一本通信原理教材，或者查看 3GPP 协议中关于 LTE/5G NR 物理层的描述，你会看到OFDM 发射机的完整链路大致如下：

比特流 → QAM 映射 → 串并转换(S/P) → IFFT → 加循环前缀(CP) → 并串转换(P/S)
→ DAC → 上变频 → 天线

这 7 个步骤，构成了 OFDM 发射机的全部核心流程。听起来步骤很多，但每一级的作用都非常明确。我们接下来逐一拆解。

2.1 QAM 映射——比特到复数符号的翻译官

二进制比特流进入发射机的第一个处理环节，就是 QAM 映射。QAM（Quadrature Amplitude Modulation，正交幅度调制）是一种同时利用载波的幅度和相位来承载信息的方式。

以 16QAM 为例，每个符号可以承载 $\log_2 16 = 4$ 个比特。一个 16QAM 星座图上有 16 个点，每个点对应一种独特的幅度-相位组合。接收端只需要判断接收信号落在了哪个点附近，就能还原出对应的 4 个比特。

import numpy as np

# 16QAM星座点生成（标准 Gray 编码）
constellation = np.array([
    -3+3j, -1+3j,  1+3j,  3+3j,
    -3+1j, -1+1j,  1+1j,  3+1j,
    -3-3j, -1-3j,  1-3j,  3-3j,
    -3-1j, -1-1j,  1-1j,  3-1j
]) / np.sqrt(10)  # 归一化，保证平均功率为1

# 比特到符号映射（Gray编码）
bit_to_symbol = {
    '00': -3+3j, '01': -1+3j, '11':  1+3j, '10':  3+3j,
    '10': -3+1j, '11': -1+1j, '01':  1+1j, '00':  3+1j,
    '11': -3-3j, '10': -1-3j, '00':  1-3j, '01':  3-3j,
    '01': -3-1j, '00': -1-1j, '10':  1-1j, '11':  3-1j,
}

从工程角度看，QAM 映射实际上完成了从信息域（比特）到信号域（复数符号）的转换。映射完成后，每个 OFDM 子载波上都会承载一个 QAM 符号。

2.2 串并转换(S/P)与 IFFT——从频域到时域的魔法

这是 OFDM 发射机中最关键的两个环节，理解它们，你就理解了一半的 OFDM。

串并转换（Serial-to-Parallel, S/P）的作用，是将串行的 QAM 符号流转换成 $N$ 路并行。$N$ 是 FFT/IFFT 的点数，在 LTE 中常取 $N=2048$（20MHz 带宽）。这意味着每 $N$ 个 QAM 符号被组织为一组，共同经历一次 IFFT 变换。

IFFT（Inverse Fast Fourier Transform，逆快速傅里叶变换）的数学本质是：从频域采样还原时域连续信号。从信号处理的角度，OFDM 系统中的 IFFT 完成了这样的操作：

\[ x[n]=\frac{1}{N}\sum_{k=0}^{N-1}X[k]\cdot e^{j2\pi kn/N},\qquad n=0,1,\cdots,N-1 \]

其中 $X[k]$ 是第 $k$ 个子载波上承载的 QAM 符号，而 $x[n]$ 是 IFFT 输出的第 $n$ 个时域采样。可以看到，每一个输出采样 $x[n]$，实际上是所有 $N$ 个子载波符号的加权和——这正是 OFDM“多载波”的数学体现。

💡 一个直观的理解：你可以把 IFFT 想象成一场”大合唱”。$N$ 个子载波符号是 $N$ 个歌手，每个人唱一个音调（频率），IFFT 就是把他们同时发出的声音混叠成一个综合的声波信号（时域采样）。接收端只需要用 FFT 做一次”声纹分离”，就能把每个人的声音（每个子载波的数据）单独提取出来。

2.3 循环前缀(CP)——OFDM 的保护间隔

IFFT 输出的 $N$ 个时域采样，经过并串转换（P/S）变成连续的采样流。在正式发送之前，还需要插入循环前缀（Cyclic Prefix, CP）。

CP的实质是：将 OFDM 符号末尾的一段采样，复制并粘贴到符号的开头。如果 $T_{CP}$ 表示 CP 的长度（以采样数计），那么发送的完整 OFDM 符号长度为 $N + N_{CP}$ 个采样。

CP 在 OFDM 中扮演着”护城河”的角色，其核心作用是对抗多径效应引起的符号间干扰(ISI)。关于 CP 的详细原理，我会在第 5 讲做深入讲解。这里先记住：CP 长度必须大于信道的最大多径时延扩展，否则保护效果会大打折扣。

2.4 DAC与上变频——从数字到射频

经过以上处理后的信号还是基带数字信号。要让它能够通过天线发射出去，还需要两个环节：

  • DAC（数模转换器）：将数字采样流转换为模拟连续时间信号
  • 上变频：将基带信号乘以载波 $\cos(2\pi f_c t)$ 或 $e^{j2\pi f_c t}$，把信号”搬移”到指定的射频频段

最终，从天线发射出去的是一个带宽为 $B$（由子载波间隔 $\Delta f$ 和子载波数 $N$ 决定）、中心频率为 $f_c$ 的射频信号。

三、时域视角 vs 频域视角——两种看待 OFDM 的方式

理解 OFDM 系统的一个重要技巧，是习惯在时域和频域之间自由切换。

  • 频域视角：在频域里，OFDM 是一组等间隔排列的窄带子载波，每个子载波承载一个 QAM 符号。信道对每个子载波的影响是独立的复数增益——这就是 OFDM 的核心优势所在。
  • 时域视角：在时域里，OFDM 是一个个带有 CP 的时域脉冲序列。每个 OFDM 符号看起来像被”矩形窗”截断的连续信号，CP 的添加使得符号边界处形成了天然的周期性。

两种视角各有优势：设计信道估计和均衡算法时，用频域视角更直观；分析峰均比(PAPR)和带外辐射时，用时域视角更方便。

四、以 LTE 20MHz为例：参数走一遍

学了理论，我们用实际系统参数来验证一下。LTE 20MHz 带宽对应的关键参数如下：

参数 数值 说明
子载波间隔 $\Delta f$ 15 kHz LTE 标准规定的子载波间隔
FFT 点数 $N_{FFT}$ 2048 20MHz / 15kHz ≈ 1333，用 2048 留余量
有用符号时长 $T_u$ 66.7 μs $T_u = 1/\Delta f = 1/15000$
CP 长度 $T_{CP}$ 4.7 μs（常规 CP） 大于典型城市多径时延扩展
一个子帧时长 1 ms 包含14个OFDM 符号（常规 CP）
总带宽 30.72 MHz 采样率 = $N_{FFT}\times\Delta f / 2$ 
# LTE 20MHz 系统参数
fft_size = 2048          # FFT点数
subcarrier_spacing = 15e3  # 子载波间隔 15 kHz
cp_length = 160         # 常规CP长度（采样）
sampling_rate = 30.72e6  # 采样率 30.72 MHz
total_symbols_per_subframe = 14

# 有用符号时长
Tu = 1 / subcarrier_spacing  # 66.7 us

# CP开销
cp_overhead = cp_length / (fft_size + cp_length)  # 约 7.2%

# 有效数据速率估算（假设1个资源块=12子载波x7符号，全带宽100个RB）
n_rb_dl = 100
n_subcarriers_per_rb = 12
modulation_order = 6  # 64QAM -> 每个符号 6 比特
coded_bits_per_subframe = n_rb_dl * n_subcarriers_per_rb * (total_symbols_per_subframe - 2) * modulation_order
# 减去2是因为每个子帧的首符号需要更长的扩展CP

五、总结

这一讲我们完整走了一遍 OFDM 发射机的 7 大环节：

  • QAM 映射：将比特映射为复数符号（频域）
  • S/P 串并转换：组织成 $N$ 路并行，准备做 IFFT
  • IFFT：将频域并行符号变换为时域采样——这是 OFDM 的核心
  • 加CP：在时域符号前插入循环前缀，对抗多径干扰
  • P/S 并串转换：将并行时域采样合并为串行流
  • DAC：数模转换，得到基带模拟信号
  • 上变频：搬移到射频频率，通过天线发射
