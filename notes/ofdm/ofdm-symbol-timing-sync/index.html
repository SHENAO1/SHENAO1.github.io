第12讲：符号定时同步——找到 OFDM 符号的“起点”

引言：时间，精确到“采样点”

想象一下，你正在收听一段话，对方说：“开——始”，但你没听到“开”字，只听到了“——始”。断章取义，意思全变了。

OFDM 接收机面临的符号定时同步✦问题，就是这么一回事。

接收端每个 OFDM 符号都有一个精确的“起点”，FFT 窗口必须恰好对准这个起点，才能正确地从时域转换到频域，还原出发端的调制数据。如果起点找偏了——哪怕只偏了一两个采样点——轻则每个子载波都多出一个相位偏转，重则相邻符号互相干扰（ISI），解调彻底失败。

这一讲，我们就来搞清楚：

  1. 符号定时偏差（STO）到底会造成什么影响？
  2. 如何利用 CP 的结构特性，设计高效的定时同步算法？
  3. Schmidl & Cox 算法为什么能成为业界标准？

一、符号定时偏差（STO）的影响分析

1.1 OFDM 符号的时域结构回顾

一个完整的 OFDM 符号在时域的结构如下：

|<-- CP（Ncp个采样）-->|<-- 有效符号（N个采样）-->|

接收端 FFT 窗口应取“有效符号”对应的 $N$ 个采样点。设正确起点为 $n_0$，实际检测到的起点为 $\hat{n}_0$，定义符号定时偏差（Symbol Timing Offset, STO）为：

\[ \varepsilon=\hat{n}_0-n_0 \]

1.2 STO的两种影响：相位旋转 vs ISI

情形一：STO 在 CP 保护范围内（$0\le\varepsilon\le N_{cp}-L_{max}$）

只要 STO 不超过 CP 扣除最大多径时延后的“安全裕量”，多径不会溢出 CP 保护范围。此时 FFT 输出的第 $k$ 个子载波为：

\[ Y_k=H_k\cdot X_k\cdot e^{j2\pi k\varepsilon/N}+W_k \]

相比于理想情况，每个子载波多了一个线性相位项 $e^{j2\pi k\varepsilon/N}$。这个相位与子载波编号 $k$ 成正比，在星座图上表现为各子载波不同程度的相位旋转，但子载波之间仍然正交，不会引入 ICI。

在信道估计阶段，这个相位旋转会被导频估计“吸收”进信道响应估计里，通常不造成严重问题——这就是 CP 带来的“容错区间”。

情形二：STO 超出安全裕量（$\varepsilon>N_{cp}-L_{max}$）

FFT 窗口开始“吃进”上一个符号的数据，相邻符号互相干扰，产生 ISI（符号间干扰），同时正交性被破坏，出现 ICI（子载波间干扰）。此时解调性能急剧恶化，基本无法可靠工作。

用一个 ASCII 示意图描述两种情形：

理想对齐：
|   CP   |<====== 有效符号 N点 ======>|

情形一（轻微偏移，仍在CP内）：
        |   CP   |<== 有效符号 N点 ==>|
     <-->（偏移量 ε < Ncp - Lmax）
     → 子载波相位旋转，无ISI

情形二（超出CP范围）：
                  |CP|<== 有效符号 N点 ==>|
       <-------ε------->（ε > Ncp - Lmax）
       → 前一符号泄漏进FFT窗口，产生ISI+ICI

1.3 STO对系统性能的量化影响

下表总结了不同 STO 范围对系统性能的影响：

STO范围 是否产生ISI 子载波相位 对BER影响
$\varepsilon=0$ 无 零 理想
$0<\varepsilon<N_{cp}-L_{max}$ 无 线性相位偏转 轻微（信道估计可补偿）
$N_{cp}-L_{max}\le\varepsilon<N_{cp}$ 有（轻微） 严重偏转 明显恶化
$\varepsilon\ge N_{cp}$ 严重 完全破坏 不可工作

二、基于 CP 的定时同步：相关法原理

2.1 CP 结构是天然的“同步锚点”

前面讲过，CP 是有效符号尾部的拷贝：

\[ r[n]=r[n+N],\qquad n=0,1,\ldots,N_{cp}-1 \]

（以符号起始点为基准，$r[n]$ 为 CP 部分，$r[n+N]$ 为有效符号尾部对应采样）

这个冗余结构意味着：只要能找到接收信号中“这两段高度相似”的位置，就找到了符号边界。

2.2 滑动相关窗口时检测

定义一个滑动窗相关度量（不含信道影响的简化形式）：

\[ P(d)=\sum_{m=0}^{N_{cp}-1}r^*(d+m)\cdot r(d+m+N) \]

\[ M(d)=\frac{|P(d)|^2}{\left(\sum_{m=0}^{N_{cp}-1}|r(d+m+N)|^2\right)^2} \]

其中 $d$ 是滑动窗的起始位置，$r(\cdot)$ 是接收信号。$M(d)$ 在符号真实起始位置附近会出现平台型峰值（宽度约为 $N_{cp}-L_{max}$）。

为什么是平台而非尖峰？

因为在“安全区间”内，CP 和符号尾部的相关性都成立，$P(d)$ 的模保持稳定。这个平台的右边沿才是真正的符号起始点（再晚就会有 ISI）。工程上通常取平台中点作为同步点，留出对称余量。

三、Schmidl & Cox算法——业界标准方案

3.1 算法背景

1997 年，Timothy Schmidl 和 Donald Cox 在 IEEE Transactions on Communications 上发表了一篇经典论文，提出了一种基于特殊训练序列（Training Symbol）的同步方案，至今仍被 IEEE 802.11a/g/n、DAB、DVB-T 等多种标准采用。

3.2 训练符号设计

Schmidl & Cox 方案要求在数据帧头部插入一个特殊的 OFDM 符号（Training Symbol），其频域特征为：只有偶数子载波上有数据，奇数子载波为零。

这样的训练符号经过 IFFT 后，时域波形具有周期性：前半段与后半段完全相同：

\[ s[n]=s[n+N/2],\qquad n=0,1,\ldots,N/2-1 \]

（可由 IFFT 下采样性质证明）

加上 CP 后，整个训练符号结构为：

|<-- CP (Ncp) -->|<-- 前半段 N/2 -->|<-- 后半段 N/2 -->|
                 ↑                   ↑
              完全相同（半周期性）

3.3 定时度量与频偏估计

步骤一：定时检测

利用训练符号内部的半周期性，计算滑动相关：

\[ P(d)=\sum_{m=0}^{N/2-1}r^*(d+m)\cdot r(d+m+N/2) \]

\[ R(d)=\sum_{m=0}^{N/2-1}|r(d+m+N/2)|^2 \]

定时度量：

\[ M(d)=\frac{|P(d)|^2}{|R(d)|^2} \]

$M(d)$ 在训练符号起始处出现峰值，峰值为 1（AWGN 信道下，信噪比足够高时）。

步骤二：整数/小数频偏粗估计

利用相关相位提取频偏：

\[ \hat{\varepsilon}_{CFO}=\frac{1}{\pi}\angle P(\hat{d}) \]

其中 $\angle$ 表示取相位角。这个估计给出的是归一化频偏，范围为 $(-1,+1)$（对应 $\pm\Delta f/2$ 的频偏范围）。

步骤三：整数倍频偏估计（第二个训练符号）

单个训练符号只能估计 $|\hat{\varepsilon}|<1$ 的小数倍频偏，超出范围需要第二个训练符号（频域内容不同）来消除模糊，完成整数倍频偏估计。

3.4 MATLAB实现示例

function [timing_offset, cfo_estimate] = schmidl_cox_sync(r, N, Ncp)
% Schmidl & Cox 定时同步与粗频偏估计
% r: 接收信号（列向量）
% N: FFT点数
% Ncp: CP长度

L = N / 2;  % 训练符号半段长度
num_samples = length(r) - N - Ncp;

P = zeros(num_samples, 1);
R = zeros(num_samples, 1);

% 计算滑动相关
for d = 1:num_samples
    seg1 = r(d : d+L-1);
    seg2 = r(d+L : d+2*L-1);
    P(d) = seg1' * seg2;            % 共轭相关
    R(d) = sum(abs(seg2).^2);       % 能量归一化
end

% 定时度量
M = (abs(P).^2) ./ (R.^2 + eps);

% 找峰值（定时起点）
[~, timing_offset] = max(M);

% 粗频偏估计
cfo_estimate = angle(P(timing_offset)) / pi;

fprintf('定时偏移估计: %d 采样点\n', timing_offset);
fprintf('归一化CFO估计: %.4f (对子载波间隔)\n', cfo_estimate);
end

参数配置示例（LTE 20MHz）：

参数 LTE 20MHz值 含义
N (FFT点数) 2048 IFFT/FFT大小
Ncp (正常CP) 144 正常CP采样数
采样率 30.72 MHz 基带采样率
训练符号半段 L 1024 N/2
最大估计CFO范围 ±7.5 kHz ±0.5×子载波间隔

四、工程实现中的关键问题

4.1 峰值检测的鲁棒性

实际信道中，$M(d)$ 的峰值会因噪声和多径而展宽、降低，出现平台型响应。工程上常用的改进策略：

1. 窗内积累（Averaging Window）： 对 $M(d)$ 在时间窗内做滑动平均，平滑噪声波动。

2. 差分检测（Differential Detection）： 计算 $M(d)-M(d-1)$，找到平台的上升沿而非峰值点，减小因平台宽度导致的偏移不确定性。

3. 能量检测门限： 设置信噪比门限，低于门限时不判定为有效帧头，避免噪声触发假同步。

4.2 多径对同步的影响

在频率选择性信道中，多径会导致 CP 和符号尾部的相关性下降（因为各径到达时间不同）。主要影响：

  • 相关峰出现“毛刺”，峰值降低
  • 定时估计偏向最强径的到达时刻

工程实践中通常在 CP 的安全裕量内稍微提前取 FFT 窗口（取平台左沿附近），确保多径不溢出 CP 范围。

4.3 5G NR中的定时同步

5G NR 的定时同步依赖主同步信号（PSS）和辅同步信号（SSS）：

  • PSS：144 个采样的 Zadoff-Chu（ZC）序列，自相关特性极佳，用于粗定时和小区 ID 检测
  • SSS：用于区分同一小区 ID 组内的 504 个小区 ID，配合 PSS 做精确定时

ZC序列的完美自相关特性（旁瓣为零）使得 5G 的定时同步比基于 CP 的方案更精确，这也是 5G 相比于 LTE 的一项关键改进。

五、与信道估计的协同设计

定时同步不是孤立的模块，它与信道估计紧密耦合：

接收信号 → [粗定时同步] → [去CP] → [FFT] → [精确定时/频偏补偿] → [信道估计] → [均衡]
              ↑                         |
              └──────────── 反馈精调 ────────────┘

粗同步： 基于 CP 相关或训练序列，精度在几个采样点以内，目的是让 FFT 窗口落在 CP 安全区间内。

精同步： 利用信道估计结果中的相位斜率，进一步估计残余 STO（对应子载波间的线性相位差），并通过时域内插或频域相位补偿消除。

这种“粗粗+精”的两级同步结构，是现代 OFDM 接收机的标准设计范式。

总结

本讲聚焦于 OFDM 接收机的第一道关卡——符号定时同步。

核心要点回顾：

  1. STO 的危害：轻则导致子载波相位旋转（CP 安全区内），重则引发 ISI+ICI（超出 CP 范围），必须精确检测和补偿。
  2. 基于 CP 的相关法：利用 CP 与符号尾部的相关性，滑动相关度量 $M(d)$ 在符号边界处出现平台峰值，是最常用的盲同步方法。
  3. Schmidl & Cox 算法：利用半周期训练符号，同时实现定时检测和粗频偏估计，在 IEEE 802.11 等标准中广泛应用。
  4. 工程实现要点：峰值检测需要鲁棒性设计，多径环境下宜取平台左沿，5G NR 采用 ZC 序列进一步提升同步精度。

定时同步是接收机的“地基”。地基打不好，后面的信道估计、均衡都是空中楼阁。
