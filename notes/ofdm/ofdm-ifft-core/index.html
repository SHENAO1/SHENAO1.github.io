第 4 讲：从傅里叶变换理解 OFDM——IFFT 为什么是核心？

引言：发射机里最关键的那一步

在上一讲，我们走过了完整的 OFDM 发射机框图：QAM✦ 映射 → 串并转换✦ → IFFT✦ → 加 CP → 并串转换✦ → DAC → 上变频✦。如果让我在这条流水线里圈出最核心的模块，我会毫不犹豫地圈住 IFFT。

为什么？因为正是这个模块，让”多载波调制✦”这件事情在工程上变成了可能。

没有 IFFT，要同时发送 $N$ 路子载波，你需要 $N$ 个独立的振荡器和混频器✦——这是上世纪模拟多载波系统✦的做法，复杂度、成本、功耗✦都高得离谱。而有了 IFFT，$N$ 路子载波的时域✦叠加，被一个算法、几百个加法器✦轻松搞定。

这一讲，我们从数学本质出发，把 DFT/IDFT✦ 与 OFDM 子载波✦的关系彻底讲清楚。理解了这个，你就理解了 OFDM 的灵魂。

一、 DFT/IDFT：离散时间信号✦的频域工具

1.1 DFT 的定义

设一个长度为 $N$ 的离散时间序列✦ $x[n]$，$n=0,1,\cdots,N-1$，其离散傅里叶变换✦（DFT）定义为：

\[ X[k]=\sum_{n=0}^{N-1}x[n]\cdot e^{-j\frac{2\pi}{N}kn},\qquad k=0,1,\cdots,N-1 \]

直觉上，$X[k]$ 就是”信号 $x[n]$ 在频率 $f_k=k/(NT_s)$ 处的复振幅”，其中 $T_s$ 是采样间隔✦。

对应地，逆离散傅里叶变换✦（IDFT）为：

\[ x[n]=\frac{1}{N}\sum_{k=0}^{N-1}X[k]\cdot e^{j\frac{2\pi}{N}kn},\qquad n=0,1,\cdots,N-1 \]

注意这里的对称性：DFT 是”时域 → 频域”，IDFT 是”频域 → 时域”，两者互为逆运算，差一个归一化系数✦ $1/N$。

1.2 OFDM 发送信号与 IDFT 的关系

现在来看 OFDM。设第 $k$ 个子载波上映射的 QAM 符号为 $X[k]$，OFDM 符号的连续时域✦信号为：

\[ s(t)=\sum_{k=0}^{N-1}X[k]\cdot e^{j2\pi k\Delta f\cdot t},\qquad 0\le t<T \]

其中 $\Delta f=1/T$ 是子载波间隔✦，$T=NT_s$ 是 OFDM 符号持续时间。

在 $t=nT_s$ 处采样，得到离散时域✦序列：

\[ s[n]=s(nT_s)=\sum_{k=0}^{N-1}X[k]\cdot e^{j2\pi k\Delta f\cdot nT_s} =\sum_{k=0}^{N-1}X[k]\cdot e^{j\frac{2\pi}{N}kn} \]

把这个和 IDFT 公式对比一下：

\[ x[n]=\frac{1}{N}\sum_{k=0}^{N-1}X[k]\cdot e^{j\frac{2\pi}{N}kn} \]

完全一致（相差一个常数归一化因子）！

结论：OFDM 符号的时域采样序列，就是频域 QAM 符号序列的 IDFT。 发送一个 OFDM 符号 = 对 $N$ 个频域复数做 IDFT。

这就是 IFFT 处于核心地位的数学根源。

二、 矩阵✦形式：看清 OFDM 的线性结构✦

用矩阵语言写出来会更清晰。定义 $N\times N$ 的 DFT 矩阵 $\mathbf{F}$，其 $(n,k)$ 元素为：

\[ [\mathbf{F}]_{n,k}=e^{-j\frac{2\pi}{N}nk},\qquad n,k=0,1,\cdots,N-1 \]

则：

  • DFT（分析）：$\mathbf{X}=\mathbf{F}\mathbf{x}$
  • IDFT（综合）：$\mathbf{x}=\mathbf{F}^{H}\mathbf{X}/N$

其中 $\mathbf{F}^{H}$ 是 $\mathbf{F}$ 的共轭转置✦（Hermitian 转置）。

OFDM 发射机发送信号：

\[ \mathbf{s}=\mathbf{F}^{H}\mathbf{X} \]

（忽略归一化系数）

在接收端，去掉 CP 后做 FFT：

\[ \hat{\mathbf{X}}=\mathbf{F}\mathbf{r} \]

对于理想信道（$\mathbf{r}=\mathbf{s}$），恢复出 $\hat{\mathbf{X}}=\mathbf{F}\mathbf{F}^{H}\mathbf{X}=\mathbf{X}$，完美还原。

这个矩阵视角非常有用——后续讲信道估计和均衡时，你会频繁看到 $\mathbf{F}$、$\mathbf{F}^{H}$ 的身影。

三、 FFT：让 OFDM 在实际中可行的加速引擎

3.1 直接 DFT 的计算量

如果暴力计算 $N$ 点 IDFT，需要：

  • 复数乘法：$N^2$ 次
  • 复数加法✦：$N(N-1)$ 次

对于 LTE 20MHz 系统，$N=2048$，直接 DFT 需要约 400 万次复数乘法——每个 OFDM 符号都要算一遍，以 LTE 14 个符号/ms 计算，每秒需要 560 亿次复数乘法。这在 2000 年代的数字芯片✦里根本不可能实时跑起来。

3.2 FFT 的分治思想

快速傅里叶变换✦（FFT），核心是 Cooley-Tukey 算法✦，利用 DFT 的对称性和周期性，将 $N$ 点 DFT 分解为两个 $N/2$ 点 DFT：

\[ X[k]=\sum_{n\ \text{偶}}x[n]W_N^{kn}+W_N^k\sum_{n\ \text{奇}}x[n]W_N^{kn} \]

其中 $W_N=e^{-j2\pi/N}$ 为旋转因子✦（Twiddle Factor）。

递归✦应用这个分解（当 $N=2^m$ 时最优），计算复杂度降为：

\[ \text{复数乘法次数}\approx\frac{N}{2}\log_2N \]

对比一下 $N=2048$ 的情况：

方法 复数乘法次数 相对倍数✦
直接 DFT $N^2=4,194,304$ 1×
Radix-2 FFT $\frac{N}{2}\log_2N\approx11,264$ 1/372×

372倍的加速！这就是为什么现代基带芯片✦能实时处理 5G NR 的 1 个 OFDM 符号（3296个子载波 @120kHz SCS）的底气。

3.3 蝶形运算✦：FFT 的基本单元

FFT 的基本计算单元是蝶形（Butterfly）运算：

a ───────┬──── a + W·b
         │
         ×W
         │
b ───────┴──── a - W·b

一个蝶形运算完成 1 次复数乘法 + 2 次复数加法。 $N=2^m$ 点 FFT 共有 $m$ 级，每级 $N/2$ 个蝶形，总计 $\frac{N}{2}\log_2N$ 次蝶形运算。

对应到硬件实现，这些蝶形运算可以流水线化——这是后续第 7 讲会详细讨论的话题。

四、 用 Python 演示 8 点 IFFT 的时域叠加

下面用代码直观展示 IFFT 是如何把频域符号合成时域波形✦的：

import numpy as np
import matplotlib.pyplot as plt


# ===== 8点IFFT演示：OFDM符号时域合成 =====

N = 8  # 子载波数✦（IFFT点数）

# 频域：每个子载波上的QAM符号（这里用简单的+1/-1示意）
X = np.array([1+0j, -1+0j, 1+0j, -1+0j, 0+0j, 0+0j, 0+0j, 0+0j])


# 方法1：直接用numpy✦的 IFFT
x_ifft = np.fft.ifft(X)


# 方法2：手动叠加各子载波时域波形（验证等价性）
t_samples = np.arange(N)
x_manual = np.zeros(N, dtype=complex)

print("子载波时域波形叠加过程：")
for k in range(N):
    if X[k] != 0:
        subcarrier_k = X[k] * np.exp(1j * 2 * np.pi * k * t_samples / N) / N
        x_manual += subcarrier_k
        print(f"  子载波 {k}: X[{k}] = {X[k]:.1f}, 频率 = {k}/{N} * fs")


# 验证两种方法结果一致
print(f"\n差异最大值：{np.max(np.abs(x_ifft - x_manual)):.2e}")  # 应接近0

# 打印时域输出
print("\n时域采样点 s[n]:")
for n in range(N):
    print(f"  s[{n}] = {x_manual[n].real:.4f} + {x_manual[n].imag:.4f}j")

运行这段代码，你会看到 4 个非零子载波（$k=0,1,2,3$）的时域分量被逐一叠加，最终形成 OFDM 符号的时域波形——这就是 IFFT 在做的事情，只不过用了高效的蝶形算法✦替代了逐项求和。

五、 LTE 参数实例：算一算 IFFT 的规模

理论讲完，来看真实系统里的数字。

以 LTE 20MHz 系统（这是目前 4G 部署最广的配置）为例：

参数 数值 说明
子载波间隔 $\Delta f$ 15 kHz 所有 LTE 配置统一
OFDM 符号持续时间 $T$ $1/\Delta f=66.67\,\mu s$ 不含 CP
系统带宽 20 MHz ——
可用子载波数 1200 资源网格中 100 个 RB × 12 子载波
IFFT 点数 $N$ 2048 $=f_s/\Delta f=30.72\text{ MHz}/15\text{ kHz}$
采样率✦ $f_s$ 30.72 MHz $=2048\times15\text{ kHz}$
FFT 级数 $\log_2N$ 11 $2048=2^{11}$
每符号蝶形运算数 $N/2\cdot\log_2N=11264$ 约1.1万次

注意：2048 点 IFFT 中只有中间 1200 个子载波承载数据/导频，两侧 848 个子载波置零（保护带），防止相邻信道干扰。

频域子载波排布（示意）：
|← 424 个零子载波 →|← 600 个子载波 →|DC|← 600 个子载波 →|← 424 个零子载波 →|
^                 ^               ^                ^                 ^
k=0              k=424           k=1024           k=1424            k=2047
                                  (DC 载波不使用)

这种“两侧归零”的处理，也是为什么 OFDM 信号在频谱上看起来像一个平顶的梯形——只有中间的资源块在发射能量。

六、 接收端：FFT 与 DFT 的对偶关系

发送端做 IFFT（频域 → 时域），接收端对称地做 FFT（时域 → 频域）。在理想信道下：

\[ \hat{X}[k]=\operatorname{FFT}\{r[n]\}=\operatorname{FFT}\{s[n]\}=X[k] \]

在实际多径信道✦下，接收信号为：

\[ r[n]=h[n]\circledast s[n]+w[n] \]

其中 $\circledast$ 表示循环卷积（这正是 CP 存在的意义，后续第 5 讲详细讲），$w[n]$ 为噪声。

FFT 之后，循环卷积变成频域逐点相乘：

\[ \hat{X}[k]=H[k]\cdot X[k]+W[k] \]

均衡只需要逐子载波做一次复数除法✦：$\hat{X}[k]/H[k]$，复杂度仅 $O(N)$——这就是 OFDM 把频率选择性均衡✦化繁为简✦的核心机制。

总结

这一讲的核心逻辑可以用一句话概括：

OFDM 的时域信号 = 频域QAM符号的 IDFT，IFFT让这个操作在计算上可行。

具体来说：

  1. 数学等价性：OFDM 符号时域采样序列就是频域符号向量的 IDFT，这是 IFFT 是核心模块的根本原因
  2. 复杂度革命：FFT 将 $O(N^2)$ 的直接 DFT 计算压缩到 $O(N\log N)$，对于 $N=2048$ 有约 372 倍的加速
  3. 矩阵视角：发送 $\mathbf{s}=\mathbf{F}^{H}\mathbf{X}$，接收 $\hat{\mathbf{X}}=\mathbf{F}\mathbf{r}$，FFT/IFFT 就是乘 $\mathbf{F}$ 或 $\mathbf{F}^{H}$
  4. 实际参数：LTE 20MHz 使用 2048 点 IFFT，每个 OFDM 符号约需 11264 次蝶形运算

下一讲（第5讲），我们会聚焦另一个关键模块——循环前缀 CP✦，理解为什么 OFDM 必须在 IFFT 输出前”头尾相接”地拼上一段尾巴，以及这件事如何让多径信道的频域均衡成为可能。
