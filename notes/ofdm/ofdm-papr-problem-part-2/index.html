第10讲：峰均比PAPR问题（下）——主流抑制技术详解

上一讲我们把 PAPR 的底裤扒了个干净——它为什么存在、有多严重、对 PA 和 ADC 有什么灾难性影响。今天趁热打铁，直接来看工程师们是怎么跟这些“峰值”掰手腕的。

PAPR 抑制技术五花八门，但可以归纳为三大流派：信号预畸变（简单粗暴但有效）、信号编码（巧妙的数学游戏）、概率类（预留后门曲线救国）。每种方法都有取舍，工程上往往是几种技术组合使用。

一、信号预畸变类——”砍掉它，然后擦干净”

1.1 限幅滤波（Clipping + Filtering）

这是最简单、最直观、也是工程中使用最普遍的 PAPR 抑制方法。思路就两个字：削峰。

设目标限幅门限为 $A_{\mathrm{clip}}$，则限幅操作为：

\[ y[n]= \begin{cases} x[n], & |x[n]|\le A_{\mathrm{clip}}\\ A_{\mathrm{clip}}\cdot e^{j\angle x[n]}, & |x[n]|>A_{\mathrm{clip}} \end{cases} \]

用通俗的话说：凡是幅度超过门限的样本，直接把幅度压到门限，相位保持不变。

用一个称为限幅比（Clipping Ratio, CR）的参数来描述削峰程度：

\[ \mathrm{CR} =\frac{A_{\mathrm{clip}}}{\sqrt{E[|x[n]|^2]}} =\frac{A_{\mathrm{clip}}}{\sigma} \]

CR 越小，削峰越狠，PAPR降得越多——但信号失真也越大。

限幅的问题在哪？ 硬削峰相当于在时域乘了一个矩形窗函数，根据傅里叶变换的卷积性质，这在频域引入了带外辐射（Out-of-Band Emission, OOB）——信号频谱会“泄露”到相邻信道，违反频谱模板。

怎么办？加滤波器。

Clipping + Filtering 就是：先削峰 → 再滤波把带外泄漏擦干净。但滤波又会引入新的峰值再生（Peak Regrowth），所以实际中常做多次限幅+滤波迭代：

原始信号 → [Clip 1 → Filter 1] → [Clip 2 → Filter 2] → … → [Clip N → Filter N] → 输出

MATLAB实现（单次限幅+滤波）：

% 限幅滤波法
function y = clip_filter(x, CR, N_fft)
    % x: 输入时域OFDM信号
    % CR: 限幅比（如CR = 1.8，即限幅电平为1.8*RMS）
    % N_fft: FFT点数（用于频域滤波）

    % Step 1: 限幅
    A_clip = CR * rms(x);
    y_clip = x;
    idx = abs(x) > A_clip;
    y_clip(idx) = A_clip * exp(1j * angle(x(idx)));

    % Step 2: 频域滤波（去除带外辐射）
    Y = fft(y_clip, N_fft);
    % 只保留有效子载波，其余置零
    N_used = N_fft; % 实际使用时根据子载波分配方案只保留有效子载波
    Y(N_used/2+2 : end-N_used/2) = 0;  % 带外置零
    y = ifft(Y);
end

优缺点总结：

维度 评价
PAPR抑制效果 ★★★★★（可控制在3~5dB范围内）
计算复杂度 ★☆☆☆☆（极低，一个if判断 + 一次FFT/IFFT）
带内失真 无法避免，EVM 会恶化
带外辐射 通过滤波可控，但峰值再生需要迭代处理
硬件可实现性 ★★★★★（FPGA/ASIC 实现非常成熟）

1.2 峰值加窗（Peak Windowing）

限幅的硬削平相当于乘以矩形窗，频谱旁瓣很高。峰值加窗的思路是用光滑窗函数（如升余弦窗、Kaiser窗）替代硬削，让削峰过渡更平滑，从而减少带外泄漏。

操作流程：检测到超过门限的峰值 → 在该位置乘以一个窗函数（宽度通常为几个样本）→ 窗与信号做乘性衰减。

相比于硬限幅，峰值加窗的带外辐射显著减小，但代价是”削得不够狠”——同一门限下 PAPR 抑制效果略逊于硬限幅。实际实现中，峰值加窗通常无需额外滤波，因为窗函数自身就是限带的。

二、信号编码类——”换个说法，峰值就小了”

2.1 选择性映射（SLM, Selected Mapping）

SLM 的核心思想很巧妙：同一个数据，用不同的“随机种子”生成多个等效的 OFDM 符号，挑 PAPR 最小的那个发出去。

具体操作：

  1. 准备 $U$ 组不同的相位旋转序列 $P_u=[P_u(1),P_u(2),\ldots,P_u(N)]$，其中 $P_u(k)=e^{j\phi_{u,k}}$
  2. 将原始频域数据 $X[k]$ 乘以每一组相位序列，得到 $U$ 个”变体”：$X_u[k]=X[k]\cdot P_u(k)$
  3. 对这 $U$ 组数据分别做 IFFT，得到 $U$ 个时域候选信号
  4. 选择 PAPR 最小的那个发送
┌─ P_1 × ─▶ IFFT ─▶ 算PAPR ─┐
│                           │
X[k] ─┬─ P_2 × ─▶ IFFT ─▶ 算PAPR ─┤
      │          ...
      └─ P_U × ─▶ IFFT ─▶ 算PAPR ─┘ ─▶ 选最小PAPR的路

为什么要用相位旋转？ 改变各子载波的相位，可以破坏“多子载波同相叠加”的条件，从而以较高概率获得低 PAPR 的候选信号。

以 $U$ 个候选信号独立选择，PAPR 的 CCDF 为：

\[ \Pr[\mathrm{PAPR}_{\mathrm{SLM}}>\gamma] =\left(1-\left(1-e^{-\gamma}\right)^N\right)^U \]

相比原始 OFDM，CCDF 被整体“压缩”了 $U$ 次——$U$ 越大，高 PAPR 概率越低。

MATLAB示例（$U=4$，QPSK）：

% SLM法PAPR抑制
N = 256;              % 子载波数
U = 4;                % 候选分支数
N_sym = 1000;         % 仿真符号数

% 生成U组随机相位旋转序列（BPSK：相位为0或π）
P = 2 * (randi([0 1], N, U) - 0.5) * 2; % 取值为+1或-1

papr_slm = zeros(N_sym, 1);
for i = 1:N_sym
    X = qammod(randi([0 15], N, 1), 16, 'UnitAveragePower', true);

    papr_candidates = zeros(U, 1);
    for u = 1:U
        X_u = X .* P(:, u);          % 相位旋转
        x_u = ifft(X_u) * sqrt(N);   % IFFT
        papr_candidates(u) = 10*log10(max(abs(x_u).^2) / mean(abs(x_u).^2));
    end

    [papr_slm(i), ~] = min(papr_candidates);
end

fprintf('SLM (U=%d): 平均PAPR = %.2f dB\n', U, mean(papr_slm));

SLM的代价：

  • 需要做 $U$ 次 IFFT（计算量翻 $U$ 倍）
  • 接收端需要知道用的是哪组相位序列，因此需要传输边信息（Side Information）——通常占用 $\log_2 U$ 个比特

2.2 部分传输序列（PTS, Partial Transmit Sequences）

PTS是 SLM 的”精细化升级版”。不是对整个频域数据旋转，而是把 N 个子载波分成 V 组”子块”，每组独立旋转，然后组合：

\[ x=\sum_{v=1}^{V} b_v\cdot x_v,\qquad b_v=e^{j\phi_v} \]

其中 $x_v$ 是第 $v$ 个分组的 IFFT 输出，$b_v$ 是复数旋转因子。优化目标：

\[ \{\phi_1^{\star},\ldots,\phi_V^{\star}\} =\arg\min\left( \max_n\left| \sum_{v=1}^{V} e^{j\phi_v}\cdot x_v[n] \right| \right) \]

子块划分策略：

策略 说明 特点
相邻分组（Adjacent） 相邻子载波分入同一组 实现简单，效果一般
交织分组（Interleaved） 子载波轮流分配到各组 PAPR抑制效果较好
伪随机分组（Pseudo-random） 随机分配到各组 效果最好

PTS vs SLM 对比：

维度 SLM PTS
核心操作 全频域相位旋转 分组旋转+最优组合
IFFT次数 U次 V次（$V \lt U$ 可达类似效果）
搜索空间 U个候选 $W^V$（若相位取 W 个值）
边信息开销 $\log_2 U$ bit $(V-1)\log_2 W$ bit
效果 中等 优于SLM（同计算量）

三、概率类方法——”预留后门，曲线救国”

3.1 预留子载波（TR, Tone Reservation）

TR 的思路非常讨巧：留出一小部分子载波不传数据，专门用来放“消峰信号”。

\[ x[n]= \underbrace{\frac{1}{\sqrt{N}}\sum_{k\in\mathcal{D}}X[k]e^{j2\pi kn/N}}_{\text{数据子载波（好信号）}} + \underbrace{\frac{1}{\sqrt{N}}\sum_{k\in\mathcal{R}}C[k]e^{j2\pi kn/N}}_{\text{预留子载波（消峰信号）}} \]

其中 $\mathcal{D}$ 是数据子载波集合，$\mathcal{R}$ 是预留子载波集合，$\mathcal{D}\cap\mathcal{R}=\emptyset$。

核心优化问题：寻找最优的 $C[k]$，使得时域峰值最小。

这本质是一个凸优化问题，可以用迭代方法求解。最简单的是TR-Clipping：检测超过门限的峰值 → 用预留子载波生成一个“反向脉冲”抵消它 → 反复迭代。

时域信号： ↑ 峰值
           |  /\
           | /  \___      ← 超过门限
      ____ |/___________ 门限 A
           |
消峰脉冲： |
           |   ]          ← 在峰值位置叠加
           |   |          ← 一个反向脉冲
           |   L
           ▼
补偿后：   |  _
           | / \___
      ____ |/___________ 门限 A

TR的优点：

  • 接收端完全无感知——预留子载波上的信号对数据子载波不造成任何干扰（正交的！）
  • 无需传边信息

TR的代价：

  • 频谱效率损失：预留子载波不能传数据，预留比例通常为 5%~10%
  • 发送功率增加：消峰信号消耗额外功率
  • 迭代收敛速度是工程挑战

3.2 时域注入（TI, Tone Injection）

TI 是 TR 的“扩星座版”。不预留空子载波，而是在某些子载波上故意把 QAM 点映射到等价星座点的扩展位置上，利用星座扩展产生的自由度来抵消时域峰值。

以 QPSK 为例：4个星座点 $\pm1\pm j$。TI 允许把某个点的幅度略微“拉伸”到更远的位置（如从 $1\to1.5$），这个拉伸量就可以用来抵消峰值。

TI不会损失频谱效率（所有子载波都可以传数据），但会略微增加平均发送功率（拉伸幅度要消耗额外功率），且实现复杂度显著高于TR。

五、工程实践：混合方案才是正解

实际系统中，几乎不存在只用一种PAPR抑制技术的情况。工程上的套路通常是：

5.1 典型基站发端方案

基带 IFFT输出
    |
    ▼
[限幅滤波（轻度）] ──▶ 粗砍大峰值，降3~4dB
    |
    ▼
[TR（预留子载波）] ──▶ 精细消峰，再降1~2dB
    |
    ▼
[DPD（数字预失真）] ──▶ 补偿PA非线性，而非PAPR本身
    |
    ▼
发送至 DAC/PA

限幅负责粗粒度削峰（计算量小、效果显著），TR负责精修（不引入带内失真），DPD则是最后一道防线——补偿PA的非线性失真而非降低PAPR。三者配合，5G基站PA的回退量可以从纯OFDM的12dB压缩到约6~7dB，PA效率翻倍。

5.2 终端方案：SC-FDMA的根本性解决

LTE/NR上行采用 SC-FDMA（又称 DFT-s-OFDM），本质上在IFFT之前加了一步 DFT 预编码：

QAM映射 → DFT(N点) → 子载波映射 → IFFT(M点，M>N) → 加CP

加上DFT这一步后，时域信号从”多载波叠加”变回了”单载波”的特性，PAPR直接降到3~5dB以内，大幅降低了手机PA的设计难度。

当然，SC-FDMA 也有代价：均衡复杂度增加（频域均衡后需要IDFT转回时域做判决），这是基站端能承受但手机端要省电的经典“链路预算”取舍。

六、小结

PAPR抑制不是一个刚性问题（没有唯一最优解），而是一个多目标优化——你要在 PAPR降低量、计算复杂度、数据率损失、EVM恶化 这几个维度之间做权衡。

实用建议：

  1. 低成本场景（WiFi/终端）： 限幅+滤波就够，简单有效
  2. 高性能场景（基站）： 限幅 + TR + DPD 三级联动
  3. 对EVM要求极严（256QAM/1024QAM）： 优先用 TR/TI 等无失真方法，避免信号预畸变引入的EVM代价
  4. 上行链路： 直接用 SC-FDMA，从根本上解决问题
