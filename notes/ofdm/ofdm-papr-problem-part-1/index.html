第9讲：峰均比PAPR问题（上）——为什么OFDM信号“峰”很多？

OFDM有一个出了名的“心病”——高峰均比（PAPR）。

做过射频硬件的同学可能会有这样的抱怨： PA（功率放大器）效率低、散热严重、成本贵……追根溯源，很多时候锅都得OFDM来背。本讲我们来深入剖析这个问题：OFDM信号的PAPR到底从哪里来？有多严重？对系统影响几何？

一、PAPR的定义：峰值有多”高”？

峰均比（Peak-to-Average Power Ratio，PAPR） 定义为：

\[ \mathrm{PAPR}=\frac{\max_{0\le t<T_{sym}} |x(t)|^2}{E\left[|x(t)|^2\right]} \]

其中 $x(t)$ 是OFDM时域信号，分子是信号功率的峰值，分母是平均功率。工程中常用dB表示：

\[ \mathrm{PAPR}_{dB}=10\log_{10}\mathrm{PAPR} \]

对于离散时域OFDM符号（IFFT输出），PAPR定义为：

\[ \mathrm{PAPR}=\frac{\max_{0\le n<N}|x[n]|^2}{\frac{1}{N}\sum_{n=0}^{N-1}|x[n]|^2} \]

直觉理解： PAPR = 20，即PAPR = 13 dB，意味着峰值功率是平均功率的20倍。PA必须能承受这个峰值，而大部分时间信号功率却远低于峰值——这就造成了极大的效率浪费。

二、PAPR从哪里来？——多载波叠加的“共振”

理解PAPR的本质，先看OFDM时域信号的表达式：

\[ x[n]=\frac{1}{\sqrt{N}}\sum_{k=0}^{N-1}X[k]\cdot e^{j2\pi kn/N},\qquad n=0,1,\ldots,N-1 \]

这是 $N$ 个子载波的叠加。每个子载波携带一个复数符号 $X[k]$（如QAM点），各自以不同的频率旋转，最终在时域叠加。

2.1 最坏情况：所有子载波同相叠加

想象极端情况：某个时刻，$N$ 个子载波全部在同一相位对齐，叠加出一个超级大峰值：

\[ |x[n]|_{\max}=\frac{1}{\sqrt{N}}\sum_{k=0}^{N-1}|X[k]|\le \frac{1}{\sqrt{N}}\cdot N\cdot A=\sqrt{N}\cdot A \]

若每个子载波幅度均为 $A$，则理论最大PAPR为：

\[ \mathrm{PAPR}_{\max}=\frac{(\sqrt{N}\cdot A)^2}{A^2}=N \]

换算成dB：$\mathrm{PAPR}_{\max}=10\log_{10}N$

子载波数 N 理论最大PAPR
64 18.1 dB
512 27.1 dB
1024 30.1 dB
2048 33.1 dB

LTE 20MHz带宽用了 2048点IFFT，理论上 PAPR可达 33 dB！当然这是极端情况，实际远不会如此，但已经足够说明问题的严重性。

2.2 实际PAPR：统计特性

实际OFDM信号的子载波符号是随机的QAM点，各子载波相位随机独立。根据中心极限定理，当 $N$ 较大时，时域样本 $x[n]$ 的实部和虚部分别趋近于高斯分布：

\[ \operatorname{Re}\{x[n]\}\sim\mathcal{N}(0,\sigma^2/2),\qquad \operatorname{Im}\{x[n]\}\sim\mathcal{N}(0,\sigma^2/2) \]

因此 $|x[n]|^2$ 服从自由度为 2 的卡方分布（即指数分布），而PAPR的统计特性用互补累计分布函数（CCDF） 来描述：

\[ \mathrm{CCDF}(\gamma)=\Pr[\mathrm{PAPR}>\gamma] \]

对于 $N$ 个独立同分布的子载波，PAPR的CCDF近似为：

\[ \Pr[\mathrm{PAPR}>\gamma]\approx 1-(1-e^{-\gamma})^N \]

以 $N=1024$，目标概率 $10^{-3}$ 为例：

\[ 1-(1-e^{-\gamma})^{1024}=10^{-3} \]

解得 $\gamma\approx 11.3\,\mathrm{dB}$。也就是说，对于1024子载波OFDM，约有0.1%的时间PAPR超过11.3 dB——这正是PA设计中必须留出“回退量（Back-off）”的根本原因。

三、高PAPR对系统的影响

3.1 对功放（PA）的影响

PA的输入-输出特性在大信号下呈现非线性（饱和区）。若OFDM信号的峰值进入非线性区：

输入功率（dBm）
     |
     |          ▲ 理想线性区
     |         /
     |        /   ← 峰值进入此区域：非线性失真
     |       /
     |------/--------- 1dB压缩点（P1dB）
     |     /
     |    /
     |___/_________

非线性失真会导致：

  1. 信号内部失真： EVM（误差矢量幅度）恶化，高阶调制（256QAM）无法解调
  2. 带外辐射（OOB）： 产生谐波和互调分量，干扰相邻信道，违反频谱模板
  3. PA效率大幅下降： 为保证线性工作，PA必须回退（Back-off）工作点

PA效率与回退量的关系（A类放大器为例）：

\[ \eta=\eta_{\max}\cdot\frac{P_{out}}{P_{sat}}\approx \frac{\eta_{\max}}{\mathrm{PAPR}_{\text{linear}}} \]

若PAPR = 10 dB，PA须回退10 dB，A类PA的理论最大效率50%直接降到约5%——这是灾难性的。

3.2 对ADC/DAC的影响

接收端ADC（模数转换器）的动态范围有限。高PAPR信号要求ADC具备足够的位宽以覆盖峰值，否则峰值被削波（Clipping），产生非线性失真。

以一个简化估算为例：

PAPR (dB) 所需ADC额外位数
6 1 bit
12 2 bits
18 3 bits

每增加1 bit量化位数，ADC面积和功耗大约翻倍。这在大规模天线（Massive MIMO）系统中尤为突出——64根天线，每根天线都需要高位宽ADC，功耗压力巨大。

3.3 EVM恶化量化分析

PA非线性可以用3阶截断点（IIP3）模型描述。当输入功率相对IIP3有 $\Delta$ dB 的回退时，三阶互调失真（IMD3）功率与信号之比为：

\[ \mathrm{IMD3}\approx -2\Delta\ \mathrm{dBc} \]

对于PAPR = 10 dB 的OFDM 信号，PA工作在平均功率时，瞬时峰值会导致IMD3强度约为 $-20\,\mathrm{dBc}$，对应EVM约为10%——对于64QAM（要求EVM < 8%）已经超标！

四、MATLAB仿真：不同子载波数下的PAPR分布

用MATLAB验证PAPR的CCDF曲线，直观感受子载波数对PAPR的影响：

% OFDM PAPR CCDF仿真
clear; close all;

N_list = [64, 256, 1024];       % 子载波数
M = 16;                         % 16QAM
N_sym = 10000;                  % 蒙特卡洛符号数
papr_threshold = 0:0.1:15;      % PAPR阈值（dB）

figure; hold on;
colors = {'b', 'r', 'k'};

for idx = 1:length(N_list)
    N = N_list(idx);
    papr_all = zeros(N_sym, 1);

    for i = 1:N_sym
        % 生成随机16QAM符号
        bits = randi([0, 1], N, 4);
        syms = qammod(bi2de(bits, 'left-msb'), M, 'UnitAveragePower', true);

        % IFFT调制（过采样4倍提高精度）
        x_fd = [syms; zeros(3*N, 1)];   % 频域零填充（4倍过采样）
        x_td = ifft(x_fd) * sqrt(4*N);  % 时域信号

        % 计算PAPR
        P_peak = max(abs(x_td).^2);
        P_avg  = mean(abs(x_td).^2);
        papr_all(i) = 10*log10(P_peak / P_avg);
    end

    % 计算CCDF
    ccdf = zeros(size(papr_threshold));
    for j = 1:length(papr_threshold)
        ccdf(j) = mean(papr_all > papr_threshold(j));
    end

    semilogy(papr_threshold, ccdf, colors{idx}, 'LineWidth', 1.5);
end

xlabel('PAPR_0 (dB)'); ylabel('Pr[PAPR > PAPR_0]');
title('OFDM PAPR CCDF (16QAM, 4倍过采样)');
legend('N=64', 'N=256', 'N=1024', 'Location', 'southwest');
grid on; ylim([1e-4, 1]);

运行上述代码，你会得到三条CCDF曲线，观察规律：

  • 子载波数越多，CCDF曲线整体右移——高PAPR出现概率更大
  • 在 $\Pr=10^{-3}$ 处，$N=64$ 约为 9.5 dB，$N=1024$ 约为 11.5 dB
  • 增加过采样倍数（4×）能更准确捕获时域峰值（Nyquist频率处的峰值不会被低估）

工程提醒： 仅用IFFT直接输出（1倍采样）计算PAPR会低估实际值约0.5~1 dB，建议至少用4倍过采样。

五、PAPR问题的严重程度：工程视角

下表总结了几种常见通信标准的PAPR典型值，以及设计应对措施：

标准 子载波数 典型PAPR
(CCDF 0.1%)		 PA回退量 应对措施
WiFi 802.11a 52 ~9 dB ~8 dB 限幅 + 前向纠错
LTE FDD 1200 ~11.5 dB ~8 dB 限幅滤波 + 数字预失真
5G NR FR1 3300 ~12 dB ~8 dB 限幅滤波 + DPD + PT-RS
WiFi 802.11ax 2048 ~12.5 dB ~8 dB TR + 限幅
LTE 上行 1200 ~3.5 dB (SC-FDMA) ~3 dB 单载波结构天然低PAPR

注意到LTE上行采用 SC-FDMA（单载波频分多址），而非OFDMA，正是为了降低UE（用户终端）的PA成本——手机电池有限，PA效率至关重要。

六、小结

本讲深入分析了 OFDM 高PAPR问题的来龙去脉：

  1. 根源： $N$ 个子载波在时域叠加，最坏情况（同相）可达 $10\log_{10}N$ dB
  2. 统计特性： 实际PAPR服从类高斯分布，通常用CCDF描述；1024子载波下，0.1%概率对应约11.5 dB
  3. 系统影响： PA非线性失真（EVM恶化 + 带外辐射）、ADC动态范围需求增加、整机功耗效率低下
  4. 工程现实： 5G NR基站PA须留 8 dB以上回退量，这直接推高了基站成本和能耗
