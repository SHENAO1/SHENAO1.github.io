Obsidian
「 Obsidian里面所有的笔记本质上都是本地电脑上的一个个独立的Markdown文件，这种安全感是任何一个云端笔记软件都给不了的。」
核心干货配置
``text
# 笔记存储方式：本地独立的Markdown文件
# 核心优势：
# 1. 数据安全与所有权：开发者跑路也不会丢失笔记，可切换任意Markdown编辑器继续使用。
# 2. 界面丝滑流畅：告别白屏和卡顿，保持专注工作心流。
# 3. AI工具绝配：原生文件系统交互能力强，特别擅长处理Markdown格式笔记。
# 基础Markdown语法：
# 一级标题：# 标题
# 二级标题：### 标题
# 加粗：加粗内容
# 删除：~~删除内容~~
# 高亮：高亮内容
# 代码块：`[语言] 代码内容 `
# 引用：> 引用内容
# 无序列表：- 列表项
# 有序列表：1. 列表项
# 插入表格/分割线：右键菜单选择
# 插入数学块：右键菜单选择
``

GitHub & GitHub Desktop
「 GitHub作为世界上最大的源代码托管平台，已经成为了计算机科学的基础设施之一了，笔记备份在里面比存在任何一个云笔记网盘里面都要安全稳定的。」
核心干货配置
``text
# GitHub Web 端操作：
# 1. 注册/登录 GitHub 账号。
# 2. 创建新仓库 (Repository)：
# - Repository name: [自定义仓库名，如 "爬爬虾的仓库"]
# - Visibility: 选择 "Private" (私有)，确保笔记不公开。
# 3. 应对 GitHub 访问慢：使用 Microsoft Store 安装 "GitHub" 工具解决网络问题。
# GitHub Desktop 客户端操作：
# 1. 下载并安装 GitHub Desktop。
# 2. 打开客户端，File -> Clone repository (克隆仓库)。
# 3. 登录 GitHub 账号。
# 4. 选择要克隆的仓库，选择本地路径，点击 Clone。
# 5. Obsidian 绑定仓库：打开 Obsidian，选择 "打开本地仓库"，指向克隆下来的文件夹。
# 6. 配置 .gitignore 文件 (在仓库根目录新建)：
# - 文件名：.gitignore
# - 内容：
# .obsidian/workspace.json
# .obsidian/app.json
# - 目的：将这些频繁修改的配置文件排除在 Git 管理之外，避免冲突。
# 7. 首次提交：在 GitHub Desktop 选中文件，填写描述，Commit -> Publish。
`

🏗️ 架构分段：Custom Attachment Location 插件 · [11:08]

「 这里我要借助一个插件，把我们的图片格式标准成一个Markdown的语法。」

  • > 核心洞察：通过配置 Custom Attachment Location 插件与 Obsidian 内部设置，解决 Obsidian 默认图片存储混乱和链接不兼容 Markdown 标准的问题，实现附件的整洁管理和跨平台无损显示。
🧩 知识解构
  • 现状痛点 ➔ Obsidian 默认图片存储方式会使根目录混乱，图片链接并非标准 Markdown 语法，导致在 GitHub 或其他 Markdown 编辑器中无法正常显示，影响笔记的通用性与可迁移性。
  • 破局路径 ➔ [安装 Custom Attachment Location 插件] → [配置附件存储路径与重命名模式] → [调整 Obsidian 内部链接设置]
  • ↳ 微观补位：关键在于将附件存储在统一的 assets 文件夹下，并以与笔记同名的子文件夹分类，同时确保使用基于当前笔记的相对路径和标准 Markdown URL 格式。
🛠️ 生产力资产
NOTE

Custom Attachment Location 插件
「 我们这种新的图片引用方式它是符合Markdown语法。这里我从Vscode试一下，我们打开它的预览可以看到图片可以正常显示，包括我们在GitHub的网页端查看笔记的时候，图片也可以正常的显示。」
核心干货配置
``yaml
# 1. 插件安装：
# - Obsidian 设置 (齿轮图标) -> 第三方插件 -> 社区插件市场。
# - 搜索 "Custom Attachment Location" (自定义附件存储位置) -> 安装并启用。
# 2. 插件设置 (Options)：
# - Markdown URL 格式 (Markdown URL format): ./assets/${notename}/${filename}
# - 附件重命名模式 (Attachment rename mode): all (全部)
# - 是否重命名附件文件 (Rename attachment file): 勾选
# 3. Obsidian 系统设置调整：
# - Obsidian 设置 -> 文件与链接 (Files & Links)。
# - 取消勾选 "使用 Wiki 链接 (Use WikiLinks)"。
# - 内部链接类型 (New link format): 选择 "基于当前笔记的相对路径 (Relative path to file)"。
# 进阶技巧：
# - 图片大小调整：在图片链接的方括号内输入数字，如 ![[图片.png|200]]。
``

``text
# 1. 手机连接电脑：
# - 使用数据线连接手机与电脑，选择 "传输文件" 模式。
# - 将电脑上的 Obsidian 笔记文件夹复制到手机存储设备中 (建议 Documents 文件夹)。
# 2. 手机端 Obsidian 配置：
# - 打开手机版 Obsidian，选择 "Open folder as vault" (打开文件夹作为库)。
# - 导航到复制进来的文件夹，点击 "使用此文件夹"，允许访问。
# - 确认 "Trust authors" (信任作者)。
# 3. 配置 GitHub 同步 (在手机端 Obsidian)：
# - 左上角菜单 -> 右上角设置 (齿轮图标) -> 找到最下方的 Git 设置。
# - 填写 GitHub 用户名 (英文名)。
# - 填写 GitHub 注册邮箱。
# - 填写 GitHub Personal Access Token。
# 4. 生成 GitHub Personal Access Token (在电脑端 GitHub 网页)：
# - GitHub 右上角个人头像 -> Settings -> Developer settings -> Personal access tokens -> Tokens (classic)。
# - Create new token (classic)。
# - Note: 为 Token 起一个名字。
# - Expiration: 选择 "No expiration" (永不过期)。
# - Select scopes: 勾选 "repository" 权限。
# - Generate token -> 复制生成的 Token (仅显示一次，务必保存)。
# - 将复制的 Token 粘贴到手机端 Obsidian 的 Git 设置中。
``
